# SoraStrike
　IMU（慣性計測ユニット）を利用して、空中で振る動作（空中打撃）を検出する、直観的な物理入力デバイスです。

### システムの概要  
　SoraStrikeは、安価な6軸加速度・ジャイロセンサとマイクロコントローラ（Arduino Uno R3またはRaspberry Pi Pico）を用いて、低コストかつ低遅延な空中打撃入力を実現する汎用入力モジュールです。
ドラムスティックや任意の棒状の物体に取り付け、空中で振って叩くような動作をすることで、PCへの入力が可能です。機械学習や画像認識を用いず、閾値ベースのアルゴリズムで動作するため、事前学習不要で応答性に優れています。

### システムの特徴
- **直観的な物理操作**  
利用者が意図した「叩く」という身体的な動作（加速度の急激な変化）を直接検知して入力に変換するため、タイミングが掴みやすく直観的な操作が可能です。

- **低コストかつ低遅延**  
安価で広く普及していて加速度センサとジャイロセンサが搭載されているMPU6050センサとマイクロコントローラで構成されています。また、マイコン上で完結する軽量なアルゴリズムにより、入力処理の遅延を最小限に抑えています。

- **高い汎用性**  
センサモジュールは小型であるため、ドラムスティックに限らず、様々なものに取り付けて独自の入力デバイスとして利用できます。

- **5種類の入力叩き分け機能**  
打撃を検知した瞬間のデバイスの傾き（Pitch角）に応じて、あらかじめ設定された5つの領域を判定し、異なる入力を送り分けることが可能です。

### システムの構成
#### ハードウェア
- 制御部: 用途に応じて Arduino Uno R3 または Raspberry Pi Pico を選択します。
  - Arduino Uno R3: 主に開発時のデバッグや、Processingで作成した可視化ツールとのシリアル通信に使用します。
  - Raspberry Pi Pico: USB HID（キーボード）として動作させ、PCに直接キー入力デバイスとして認識させる場合に使用します。

- センサ部: 6軸加速度・ジャイロセンサ MPU6050 を搭載したブレイクアウトボード（GY-521）。I2C通信で制御部と接続します。

- 表示部: システムの初期化状態やエラー、正常動作を通知するためのインジケータLED。  
  
#### ソフトウェア
- ファームウェア (`src/Raspberry Pi Pico/Arduino/SensorFirmwarePico`または`src/Arduino Uno R3/Arduino/SensorFirmware`)  
  　Arduino IDEで開発。センサデータの取得、ノイズフィルタリング、姿勢推定、打撃検出、およびPCへのデータ送信（シリアル通信またはキーボード入力）を行います。

- システム可視化ツール (`/src/Arduino Uno R3/Processing/SystemVisualizer/`)  
　Processingで作成されたPCアプリケーション。制御部からシリアル通信で送信される姿勢データ（クォータニオン）や動加速度、システムの状態をリアルタイムで3Dモデルに描画し、動作のデバッグやパラメータの可視化を行います。

### システムの動作原理
本システムは、以下のステップで空中打撃を検出し、入力に変換します。

1. データ取得とフィルタリング  
MPU6050から加速度と角速度を周期的に取得し、中央値フィルタでノイズを除去します 。

2. 姿勢推定  
Madgwickフィルタを用いてセンサデータを統合し、デバイスの姿勢をクォータニオンとして正確に推定します 。

3. 動加速度の計算  
推定した姿勢から重力加速度成分を除去し、物体の純粋な動き（動加速度）を算出します 。

4. 空中打撃の検出  
「待機（`Idle`）」「振り下ろし（`LowSwing`）」「急停止（`Stopping`）」の3つの状態を定義した状態遷移モデルに基づき、空中打撃を検出します。

    - センサ座標系のZ軸方向への振り下ろし動作（特定の角速度と負の加速度）を検知すると`LowSwing`状態に移行します。

    - その後、Z軸の正方向に急激な加速度（急停止または反発）を検知すると`Stopping`状態となり、打撃入力として確定します。

5. 入力の分類  
打撃が確定した瞬間のPitch角（左右方向の傾き）に応じて、あらかじめ設定された5つの閾値と比較し、どの入力かを判定します。


### 実験の結果
本デバイス（スティック入力）と一般的なキーボード入力のパフォーマンスを、指定されたターゲットを連続で選択するタスクで比較しました。

- **設計者によるテスト**:
システムに習熟した設計者が操作した場合、スティック入力はキーボード入力と比較して平均反応時間およびタスク完了時間が約17.4%短縮され、より高速な入力が可能であることが示されました。

- **初見の実験協力者によるテスト**:
初めてデバイスを使用する実験協力者10名の平均では、スティック入力はキーボード入力よりもタスク完了時間が約19%長くなり、パフォーマンスが低下しました。これは、操作への習熟が必要であることを示唆しています。
  
  
**結論**:  
アンケートでは「直感的で楽しい」と高く評価されており、これらの結果から、本システムは練習による習熟によって、従来の入力デバイスよりも効率的で有効な入力手段となる可能性があると考えられます。

### 今後の展望
- #### 磁気センサの導入による検出精度の向上
  現状の6軸センサではヨー角（水平方向の回転）のドリフトが避けられないため、振り下ろし方向がセンサのZ軸に限定されています。9軸センサ（磁気センサ付きIMU）を導入して絶対方位を取得し、地球座標系を基準に動作を検出することで、デバイスの向きに依らない、より自由でロバストな打撃検出を目指します。
